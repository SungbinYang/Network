## NIC, L2 Frame, LAN 카드 그리고 MAC 주소

- NIC는 흔히 LAN카드이다.
- 유/무선 NIC가 있지만 굳이 구별하지 않고 NIC라고 흔히 부른다.
- NIC는 H/W이며 MAC주소를 갖는다.

앞으로 LAN카드라는 표현보다는 NIC라고 표현하는게 있어보인다. host는 컴퓨터에 N/W가 연결되면 host라고 부른다. 이게 그림을 그릴때도 주로 네모로 그리는데 이 네모에 진한 점을 찍고 직선형태로 선을 있는데 이때 진한 점이 NIC이고 이 NIC가 어디와 연결되었을 때 직선형태의 선을 긋는다. 그럼 host에 NIC는 1개만 있을까? 흔히 노트북을 생각하면 유선 NIC와 무선 NIC가 존재하듯이 NIC는 여러개가 존재할 수 있다.

그럼 LAN이라는 말도 나오는데 LAN은 무엇일까? 우리가 네트워크 규모를 애기할때 가장 큰 순으로 WAN -> MAN -> LAN순으로 나타내는데 즉, 네트워크 규모의 가장 작은 규모가 LAN이다. 예를 들면 우리 집의 공유기에는 휴대폰, PC, TV등 많은 host들이 연결되어 있는데 이 공유기와 여러 호스트들을 합쳐서 LAN이라고 한다.

MAC주소라는 것은 H/W NIC에 부여된다. 우리가 앞서서 패킷이라는 말을 했는데 이 패킷은 L2에서 언급하는 인터넷에서의 단위이고 L2수준 자체에서의 단위가 있는데 그게 바로 Frame이다. N/W 인프라 구조가 있다면 이 환경에서 유통되는 단위가 Frame이다. 정확히는 데이터의 단위라고 한다.

우리가 유선이라 말하는 이더넷 네트워크는 L2 Frame Data라고 단위이고 이것의 크기는 1514byte정도 된다. 그만큼 매우 작다. 또한 속도는 1Gbps라고 하는데 여기서 주의할 점이 b가 대문자가 아니라 소문자로 표기해야한다.

그런데 가끔 10Gbps가 있는데 이때는 케이블이 구리선이 아니라 광케이블일 경우 이정도 속도로 나온다.

## L2 스위치에 대해서

### L2 Access Switch

- End-Point와 직접 연결되는 스위치
- MAC 주소를 근거로 스위칭

우리가 앞서서 NIC에 대해 이야기를 했다. 이게 웃긴게 뭐냐면 이 NIC 하나당 케이블 한 가닥이라고 생각하면 된다. 아무튼 L2 스위치는 End-Point와 직접 연결되는 스위치라고 하는데 이 때 L2 스위치라고 안하고 L2 Access 스위치라고 한다. 그래서 DoD기반으로 네트워크를 분류할 때 1계층을 N/W Access 계층이라고 하는데 그래서 N/W Access단 즉, PC같은 End-Point하고 LAN케이블이 직접 연결되는 스위치를 L2 Access 스위치라고 한다. 그리고 L2스위치이다 보니까 MAC주소를 가지고 스위칭을 한다. 근데 이때 스위치의 단자 1개를 인터페이스라고 하는데 다른 말로는 port라고도 한다. 그래서 포트가 24개면 24포트 허브라고도 한다.

여기서 중요한게 뭐냐면 NIC를 보면 위에 LED가 붙어있는데 LED가 주황색이거나 노란색이면 N/W가 충돌이 나서 작동이 안된다는 것이고 이게 만약 녹색으로 깜빡깜빡거리면 네트워킹을 잘 하고 있다는 의미이다. 이때 네트워킹이 잘 하고 있다고 안하고 고급스럽게 Link-up이라고 하고 주황색불이 들어오면 Link-down되었다고 의미한다.

주의할게 L2스위치에 나와 라우터같은데 연결이 되면 그때 그선을 up-link라고 한다. Link-up과 유사하니 조심하자.

다시 돌아와서 보통 end-point와 L2 Access Switch와 직접 연결이 되고 이 스위치와 라우터가 직접 연결되는것이 아니라 또 다른 스위치를 거쳐서 연결이 되는데 이 스위치를 L2 Distribution Switch라고 한다.

### L2 Distribution Swtich

- 쉽게 생각하면 L2 Access Switch를 위한 스위치이다.
- VLAN(Virtual LAN)기능을 제공하는 것이 일반적이다.

L2 Access 스위치는 L2 Distribution Swtich와 연결되는것이 일반적이고 이 스위치는 고성능이다.

그러면 이 스위치는 건물에 몇개 존재할까? 보통은 사무실 1개당 L2 Access 스위치가 존재하고 한 층마다 L2 Distribution 스위치가 존재하고 건물 1개마다 라우터 1개가 존재한다.

![](https://velog.velcdn.com/images/bini/post/589a654d-ddda-4f9d-80f0-e4458c4e85d2/image.jpg)

위와 같은 그림이 있을 때 1번 PC에서 2번 PC로 데이터 전송할때는 L2 Access 스위치만 거치면 처리가 가능하지만 3번에서 5번으로 데이터 전송시에는 L2 Access -> L2 Distribution -> L2 Access로 거쳐서 데이터 전송이 가능하고 다른 지역의 누군가에게 데이터 보낼 시에는 라우터를 거친다.

## LAN과 WAN의 경계 그리고 Broadcast

- Broadcast 범위를 생각해보자.
- Broadcast 주소라는 매우 특별한 주소가 존재 (MAC, IP 모두 존재)
- 논리적인것인지 아니면 물리적인것인지로 구분하는 방법도 있다.
- 일단 MAN은 나중에 생각

네트워크 애기를 할때 굉장히 중요한 이슈중에 하나가 바로 Broadcast이다. 그대로 번역하면 방송인것이다. Broadcast주소가 있는데 말 그대로 방송주소가 되는데 이 방송주소가 중요하고 여러가지 범위로 응용되어 활용되기도 하는데 문제는 뭐냐면 Broadcasting이라는것은 효율을 매우 떨어트리는 놈이다.

> 이 BroadCast의 정반대가 Unicast인데 이 Unicast는 한 곳을 콕 찝어서 애기하는 것이다.

예를 들어 두 사람이 애기를 소곤소곤하고 있다고 해보자. 그러면 근처의 주위사람들에게는 이 소리가 들리지 않을 것이다. 만약 이 소리가 들린다면 주위사람들에게 소음을 발생해 하고 있던일에 집중을 하지 못할 것이다. 즉, 브로드캐스트는 소음이다. 그래서 브로드캐스트는 꼭 필요할때만 써야하는 제한적인 통신방법이다. 그러다보니 브로드캐스트의 범위를 최소화하는게 중요하다.

Broadcast주소라는 특별한 주소가 존재하는데 MAC주소같은 경우 기본적으로 2진수로 보았을 때 전부 값이 1이면 브로드캐스트이다. MAC주소는 48bit 주소체계를 사용하는데 예로 FF-FF-FF-FF-FF-FF이 브로드캐스트이다.

옛날에 네트워크 개발시에는 '같다' 혹은 '틀리다'라는 비교연산을 AND연산을 통하여 사용했는데 FF라는것은 모든것에 다 해당시키겠다는 것이다.

주소라는것에는 출발지 주소가 있고 목적지 주소가 있는데 목적지 주소가 broadcast이면 전부 받으라는 의미이다. 즉, 소음을 발생할때 주위에 있는 모든 사람에게 소음을 발생시키겠다는 것이다.

![](https://velog.velcdn.com/images/bini/post/589a654d-ddda-4f9d-80f0-e4458c4e85d2/image.jpg)

LAN환경에서 통신을 한다고 했을때 브로드캐스팅을 한다는 것은 여러의미가 있을 수 있는데 위의 그림처럼 1번이 브로드캐스팅을 한다고 하면 2,3번뿐만 아니라 4,5번, router까지 전부 퍼지는것을 의미한다. 이렇게 브로드캐스트가 일어나면 이 브로드캐스트가 일어나는 동안 나머지 host들은 통신을 못한다. 그래서 범위를 최소화 하는게 중요하다.

LAN에서 보내는 단위가 Frame인데 이 Frame은 header + data영역으로 나눠지는데 이때 header에 출발지주소와 목적지주소를 가지고 있다 택배 송장을 생각하면 쉬울것이다. 이때 목적지주소가 전부 1이면 브로드캐스트를 의미하고 네트워크 성능이 매우 느려진다.

그래서 이것을 방지하기 위해 L2 Distribution Switch에서 다른곳으로 퍼트리지 않고 묶어둔다고 하면 브로드캐스트는 1~3번으로 끝나서 브로드캐스트를 방지할 수 있지만 만약 꼭 보내야하는 브로드캐스트이면 반응을 하지 못하는 문제가 발생해 좀 잘 생각해야하는 문제이다. 대부분 IP기반 네트워크를 사용하기 때문에 브로드캐스팅범위는 보통 IP주소범위정도로 설정한다.

![](https://velog.velcdn.com/images/bini/post/f3647ae7-7ce8-408a-980f-1ba0be64808d/image.png)

그리고 네트워크 규모문제인데 OSI 7 Layer에서 L1~L2를 지칭하는 Ethernet은 H/W이고 H/W는 물리적이기 때문에 물리적 네트워크라고 칭한다.
하지만 L3부터는 Logical 네트워크라고 하는데 인터넷같은 경우가 그렇다. L3부터는 S/W이고 S/W는 Logical = Virutal이기에 흔히 논리적 네트워크라고 불린다.

LAN은 L1~L2를 칭하는데 물리적 네트워크 눈에 보이는 네트워크를 LAN으로 칭한다. 흔히 광케이블, 인터페이스등이 있다. 인터넷은 WAN인데 눈에 보이지 않은 가상 네트워크를 칭한다.

그림을 통해 보면 WAN은 LAN 위에 있는거라고 보면 된다.
